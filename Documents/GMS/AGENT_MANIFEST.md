# GMS — Agent Manifest (Global Management System)
> Machine-consumable project manifest for automating build/scaffold/test/deploy tasks.
> Save as `AGENT_MANIFEST.md` in repository root.

## 1. Project goal (single sentence)
Build and operate a production-grade backend for GMS (Global Management System) — a multi-tenant school management platform — using Spring Boot, PostgreSQL, JWT auth, and the provided database schema, JPA entities, and OpenAPI contract. The agent MUST avoid inventing new domain logic; only implement scaffolding, wiring, tests, migrations, controllers/DTOs, and infra automation per explicit tasks below.

## 2. Required artifacts (already present / provided)
- PostgreSQL DDL (full DDL with ON DELETE CASCADE) — file: `sql/gms_schema_full.sql`
- OpenAPI 3.0 YAML skeleton — file: `openapi/openapi.yaml`
- JPA entity Java snippets and production scaffold — document: `docs/GMS_JPA_Entities.txt` (contains entities, converters, security skeleton)
- Roadmap & module plan — `docs/roadmap.md`

> **Agent responsibility:** Use the above artifacts as the source of truth. Do not change business rules unless instructed.

## 3. Allowed agent actions (policy)
1. **Must do:**  
   - Create controllers + DTOs (CRUD+bulk endpoints) that map to OpenAPI & entities.  
   - Create/revise repository interfaces, service interfaces, and service skeletons that call repositories.  
   - Implement attribute converters for JSONB and arrays (use recommended libraries).  
   - Generate Flyway migration files from the DDL and add them under `db/migration/`.  
   - Wire security (JWT provider, filter), password hashing, and role mapping (user_roles).  
   - Add unit & integration tests (JUnit + Mockito + Testcontainers).  
   - Create Postman / Newman collection or curl examples for acceptance tests.  
   - Add CI workflow files (GitHub Actions) to build, run tests, lint, and run migrations in CI.  
   - Create Dockerfile(s) and `docker-compose.yml` for local dev (app + Postgres + Redis if needed).  
   - Produce clear commit messages and open PRs with tests passing.

2. **Must NOT do (unless explicitly instructed):**  
   - Invent new business logic, change domain rules, or choose different fields than those in the DDL/OpenAPI/JPA docs.  
   - Make production security decisions (e.g., expose keys) outside `.env` / secrets management.  
   - Push to protected branches or deploy to production without an explicit deploy command.
